Payment Card Industry Data Security Standard (PCI DSS)

Comprehensive Guide to PCI DSS Compliance

What is PCI DSS?

The Payment Card Industry Data Security Standard (PCI DSS) is a set of security standards developed to ensure that all companies that accept, process, store, or transmit credit card information maintain a secure environment. The standard was created to increase controls around cardholder data to reduce credit card fraud. Compliance with PCI DSS is mandated by the card brands and administered by the Payment Card Industry Security Standards Council.

Key Facts About PCI DSS

  • Governing Body: Payment Card Industry Security Standards Council (PCI SSC)
  • Established: December 2004
  • Current Version: PCI DSS v4.0 (released March 2022)
  • Applies To: Any entity that stores, processes, or transmits cardholder data
  • Key Focus: Protection of payment card data and secure processing environments

PCI DSS Compliance Requirements

PCI DSS is organized into six major control objectives, containing a total of twelve requirements:

Build and Maintain a Secure Network and Systems

Requirement 1: Install and maintain network security controls

Network security controls (firewalls and other network security technologies) protect cardholder data by restricting inbound and outbound network traffic to/from the Cardholder Data Environment (CDE).

  • Implement network security controls between wireless networks and the CDE
  • Document network security control configurations
  • Review network security control rule sets at least every six months
  • Configure network security controls between public-facing web servers and the CDE

Requirement 2: Apply secure configurations to all system components

Default settings for system components often favor ease of use over security, creating potential security vulnerabilities that can be exploited by attackers.

  • Maintain an inventory of system components in the CDE
  • Implement only one primary function per server
  • Change default passwords on all system components
  • Document and implement security configuration standards for all system types
  • Manage all changes to system components using change control procedures

Protect Account Data

Requirement 3: Protect stored account data

Protection methods such as encryption, truncation, masking, and hashing are critical components of cardholder data protection.

  • Store only the minimum amount of cardholder data necessary for business purposes
  • Do not store sensitive authentication data after authorization
  • Mask the primary account number (PAN) when displayed
  • Render PAN unreadable using strong cryptography
  • Document and implement procedures to protect any keys used for encryption

Requirement 4: Protect cardholder data with strong cryptography during transmission

Encryption of cardholder data during transmission over public networks prevents unauthorized access to sensitive information.

  • Use strong cryptography and security protocols to safeguard cardholder data during transmission
  • Only send PAN over secure channels, never via unprotected channels like email or chat
  • Maintain an inventory of all trusted keys and certificates

Maintain a Vulnerability Management Program

Requirement 5: Protect all systems and networks from malicious software

Anti-malware solutions protect systems and networks from current and evolving malware threats.

  • Deploy anti-malware solutions on all applicable systems
  • Ensure anti-malware programs are kept current and perform periodic scans
  • Implement a process to actively detect and protect against malware

Requirement 6: Develop and maintain secure systems and software

Security vulnerabilities in systems and applications provide opportunities for attackers to access sensitive information.

  • Establish a process for identifying and assigning risk to security vulnerabilities
  • Develop software securely following industry-accepted processes
  • Maintain an inventory of custom and third-party software components
  • Patch all software components in a timely manner
  • Train developers in secure coding techniques and application security

Implement Strong Access Control Measures

Requirement 7: Restrict access to system components and cardholder data

Access control systems limit access to cardholder data and systems to only authorized personnel.

  • Define access roles and privileges based on job classification and function
  • Follow the principle of least privilege for all access assignments
  • Review all user access rights every six months

Requirement 8: Identify users and authenticate access to system components

Identifying and authenticating users ensures only authorized individuals have access to sensitive data and systems.

  • Implement multi-factor authentication for all non-console administrative access to the CDE
  • Assign unique credentials to each individual with access to system components
  • Secure user authentication credentials during transmission and storage
  • Implement strong password policies for all users

Requirement 9: Restrict physical access to cardholder data

Physical security prevents unauthorized individuals from accessing, viewing, or removing devices, data, or systems.

  • Restrict physical access to locations with cardholder data
  • Implement procedures to identify and authorize visitors
  • Control physical access to sensitive areas for personnel
  • Protect physically accessible devices from tampering or substitution

Regularly Monitor and Test Networks

Requirement 10: Log and monitor all access to system components and cardholder data

Logging mechanisms and the ability to track user activities are critical for effective security management and forensic investigations.

  • Implement audit trails to link access to system components to individual users
  • Configure audit logs to record all critical events
  • Secure audit trails so they cannot be altered
  • Review logs and security events daily
  • Retain audit trail history for at least one year

Requirement 11: Test security of systems and networks regularly

Regular testing of security systems and processes is essential for identifying vulnerabilities and security weaknesses.

  • Maintain an up-to-date inventory of authorized wireless access points
  • Perform quarterly vulnerability scans
  • Conduct internal and external penetration testing at least annually
  • Deploy network intrusion detection/prevention systems
  • Implement a methodology for penetration testing

Maintain an Information Security Policy

Requirement 12: Support information security with organizational policies and programs

Strong security policies communicate the company's security posture to personnel and establish expectations for protecting cardholder data.

  • Maintain a comprehensive security policy that addresses all PCI DSS requirements
  • Conduct annual risk assessments to identify critical assets, threats, and vulnerabilities
  • Implement a formal security awareness program for all personnel
  • Screen potential personnel prior to hire
  • Ensure security policies and procedures clearly define information security responsibilities
  • Manage service providers with policies and procedures
  • Document and implement an incident response plan

PCI DSS Compliance Levels

The compliance validation requirements differ based on the volume of credit card transactions processed annually:

Level Transaction Volume (Visa/Mastercard Definition) Validation Requirements
Level 1 More than 6 million transactions annually
  • Annual Report on Compliance (ROC) by Qualified Security Assessor (QSA)
  • Quarterly network scan by Approved Scanning Vendor (ASV)
  • Attestation of Compliance form
Level 2 1 to 6 million transactions annually
  • Annual Self-Assessment Questionnaire (SAQ)
  • Quarterly network scan by ASV
  • Attestation of Compliance form
Level 3 20,000 to 1 million e-commerce transactions annually
  • Annual Self-Assessment Questionnaire (SAQ)
  • Quarterly network scan by ASV
  • Attestation of Compliance form
Level 4 Less than 20,000 e-commerce transactions annually and all other merchants up to 1 million transactions annually
  • Annual Self-Assessment Questionnaire (SAQ) - varies by merchant category
  • Quarterly network scan by ASV (for some SAQ types)
  • Attestation of Compliance form
